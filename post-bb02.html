The Transformer: How Language Becomes Mathematics

Before entering the mathematics, it is worth fixing a distinction that changes how the problem is viewed. The black box is not everywhere, even though the expression circulates in public debate as if it described the entire system, as if language models were fundamentally mysterious by nature. They are not. The architecture is fully known, the training algorithm is fully known, inference is describable as a sequence of elementary mathematical operations. Opacity resides in one precise and circumscribed point: in the relation between the learned parameters and the functions those parameters collectively compute. Everything else is accessible. The box has an inside that can be examined.

This series attempts to do so in a structural, not metaphorical, way, following the flow of the signal from input to output, operation by operation. In this second post I start from the beginning, from how language enters the system and how the system processes it.

The first thing that happens when we write something to a language model is that our text is broken into elementary units called tokens. A token can be thought of as a word, but the correspondence is not exact. Contemporary tokenizers split text into sublexical units: sometimes a full word, sometimes a root, sometimes a group of characters. The sentence “I think” may become [“I”, “think”] or [“I”, “th”, “ink”]. What matters is that each token receives an integer, and from that moment on the model no longer works on text but on sequences of numbers.

These numbers are transformed into vectors, that is, into points in a high-dimensional geometric space. Each token becomes a position in this space. This is where language enters mathematics: not as a symbol to interpret, but as a coordinate to compute. From this moment onward, all operations of the model are geometric operations on these points. There is no semantics, only mathematical logic.

The heart of the transformer is the attention mechanism. The name is misleading if taken literally, because it suggests something similar to human attention, a selection, a choice of where to look. It is not. What happens is a weighted average. For each token in the sequence, the model computes how compatible each of the other tokens is with it, and then constructs a combination of all of them, with weights proportional to this compatibility. No token is discarded. None is chosen. All contribute, with different weights. The domain is mathematical and probabilistic.

This distinction, between weighted average and selection, matters. Those who speak of attention as if the model decided what to focus on are using mentalistic language that does not correspond to the mathematical operation. The model decides nothing; it computes. The attention weights emerge from a dot product between vectors, normalized by an exponential function. It is a continuous, differentiable operation, with no discrete choice of any kind.

After attention, each vector passes through a feed-forward network: two linear projections with a nonlinearity in between. The first expands the vector into a larger space, the second brings it back to the original dimension. This is where the model can express combinations of information that would be inaccessible in a smaller space. Then the vector returns enriched to the next layer, thanks to residual connections that preserve the original signal instead of overwriting it.

This sequence, attention followed by feed-forward, repeats for dozens of stacked layers. Each layer receives the vectors from the previous one and transforms them. At the end, the model produces a probability distribution over the entire vocabulary: for each possible next token, an estimate of how likely it is to follow the input sequence. The output the user reads is a sample from this distribution.

This is the point I find most interesting. Each of these operations, taken individually, is mathematically elementary. Embedding is a table lookup. Attention is a weighted average. Feed-forward is a composition of two linear projections. Layer normalization is statistical standardization. None of these operations, by itself, does anything semantically relevant. The complex behavior emerges from the composition.

This has an important consequence for how we think about the model. There is no layer that understands syntax, one that tracks referents, and one that knows facts. When interpretability researchers look for such components, they find activation patterns distributed across many layers and many attention heads, not functions localized in specific components. Semantics does not inhabit any precise place in the model. It emerges from composition as a global property, not a local one.

Those who look for where thought resides in the model make the same mistake as those who once looked for the homunculus in the brain. There is no where. There is a composed function whose aggregate behavior, on certain inputs, superficially resembles the result of a thought. The resemblance is real. The identity is not.

In the next post I will enter the training process: how the model learns its parameters, what it means to minimize a loss function over billions of examples, and why the solution found is one among many possible, not the best in any absolute sense.
